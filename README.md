# CSV parser is not easy

RFC4180準拠の厳密CSVパーサを **Delphiを基準** に多言語実装し、
論理行（改行含む）、巨大CSVのストリーム処理、Excel互換問題、
ETL設計、DBAの責任境界、監査・再現性までを体系的に整理した技術資料です。

## このリポジトリについて

CSVは「単なるテキスト」ではありません。
人間の運用・Excel・巨大データ・ETL・DB制約が絡んだ **プロトコル** です。

本リポジトリでは以下を扱います。

- RFC4180準拠の厳密CSVパーサ（状態遷移）
- 論理行（改行を含むCSV）の正しい扱い
- 巨大CSVのストリーム処理とメモリ設計
- Excel互換という現実との戦い
- ETL設計論（Extract / Transform / Load）
- DBA視点での責任境界・監査・再現性・データ品質

## 対象読者

- 業務系・基幹系エンジニア
- DBA / データエンジニア
- CSV事故を一度でも踏んだことがある人
- 「Excelで開けるCSV」に苦しめられた人
